# DOMAIN

Пользователь работает с данными о продажах товаров и формирует текстовый сводный отчет. Данный проект предназначен для обучения сотрудников и не предназначен для операционной деятельности. У менеджера, есть возможность формировать данные и ожидаемый результат.

## Зачем нужен этот документ

Этот документ объясняет, с какими сущностями работает модуль и какую задачу он решает.
По нему можно понять смысл кода еще до чтения реализации.
Что считается входными данными

Источник данных — текстовый файл в кодировке UTF-8. Каждая строка описывает одну запись о продаже.

## Основные сущности

Продажа: Одна запись о товаре, включающая: название товара; категорию; цену за единицу; количество.

Категория: Группа товаров, по которой нужно считать суммарную стоимость продаж. Примеры: drinks, bakery, fruit

Сумма по позиции: Стоимость одной записи: unit_price * quantity

Общая сумма: Сумма всех корректных записей. При необходимости к ней может применяться скидка в процентах.

## Отчет

Текстовый результат обработки, который содержит:

    заголовок;
    суммы по категориям;
    итоговую сумму.

## Use-case

Пользователь должен иметь возможность:

    получать полную информацию о продажах;
    работать только с корректными данными о продажах;
    получать итоговую стоимость всех продаж;
    выделять продажи с суммой не ниже заданного порога;
    получать сводку продаж по категориям товаров;
    получать текстовый отчет по результатам обработки;
    сохранять подготовленный отчет для дальнейшего использования.

## Ограничения области

В этой версии:

    не учитываются налоги;
    не поддерживаются возвраты;
    не рассматривается многовалютность;
    не ведется учет клиентов и склада;
    нет интеграции с другими системами компании.
