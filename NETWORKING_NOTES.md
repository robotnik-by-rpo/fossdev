## Локальынй запуск с 127.0.0.1
Каждый сервис слушает адрес 127.0.0.1 и использует разные порты. Все процессы имеют доступ к одному и тому же хосту, поэтому сервисы могут общаться через 127.0.0.1:порт.

## Неудачный запуск с контейнера с 127.0.0.1
Каждый dokcer контейнер имеет изолированную среду, localhost внутри контейнера указывает на сам контейнер, а не на другие контейнеры.
Когда в одном контейнере пытаются обратиться к localhost он пытается найти сервис внутри себя.

## Успешное взаимодействие через имена сервисов
Docker дает DNS-сервер для пользовательских сетей. Когда подключаетесь к одной сети Docker регистрирует названия контейнеров в DNS. Контейнер может обращаться к другим контейнерам по их именам. DNS в свою очередь преоьразует эти названия в правильный ip адрес контейнера

## Взаимодействие через хост с использованием 'host.docker.internal'
Используется, когда нужно сервисам в контейнерах связаться с сервисами, которые работают локально не в контнейнерах
